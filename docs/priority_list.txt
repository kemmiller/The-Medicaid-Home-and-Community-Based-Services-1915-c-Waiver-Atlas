EXTRACTOR ARCHITECTURE                                  (2026-06-08)
======================================================================
Four tiers, in design order (oldest → newest), each used as a
progressive fallback for the previous tier when its input is
unavailable or unreadable. The pipeline currently runs Tiers 1–3
(see PIPELINE STATUS); Tier 4 (MISC) is implemented but NOT yet
wired into the orchestration code.

  Tier 1  HTML extractor       (extractors/html_extractor/ +
                                 extractors/secondary_extractor/)
          Most comprehensive. Reads .htm/.html with form fields.
          Covers all sections including Appendix E + I.

  Tier 2  Text extractor       (extractors/text_extractor/)
          Plain-text fallback. Mostly a subset of HTML core; cannot
          parse Appendix E/I form fields.

  Tier 3  PDF AcroForm extractor (extractors/pdf_acroform_extractor/)
          Reads AcroForm field tokens (svgeninfo:*, svapdxXY_*).
          Narrow set of authoritative merged radios.

  Tier 4  MISC PDF extractor    (extractors/misc_extractor/)
          Visual-geometry fallback for FLATTENED PDFs where AcroForm
          tokens are absent. Calibrated on CO.0006.R06.00. Highest
          variable count (151) because of full B-1/A-7 table walks.

PIPELINE STATUS                                         (2026-06-08)
======================================================================
Orchestration entrypoints:
  run/run_extraction.py        — runs HTML or Text per --mode flag
  merge/merge_extractions.py   — field-level merge of HTML + Text;
                                 AcroForm optional via --pdf_csv
  scripts/misc_coverage_analysis.py — STANDALONE (not in pipeline);
                                 only used for MISC vs baseline
                                 benchmarking

What's wired:
  HTML extractor       → run_extraction.py --mode html
  Text extractor       → run_extraction.py --mode text
  Merge HTML + Text    → merge_extractions.py (default)
  AcroForm merge       → merge_extractions.py --pdf_csv (manual)

What's NOT wired (GAPS):
  MISC                 → not called by any pipeline script
  AcroForm auto-detect → must be invoked manually
  No end-to-end orchestrator producing the final `1915c-waiver-
    level.csv` referenced in older TODOs; that file appears to be
    externally generated

Target pipeline state after Phase 3 (see ACTIVE TASK BOARD):
  run_all.py (new) orchestrates HTML → Text → AcroForm → MISC and
  emits a single merged CSV at outputs/1915c-waiver-level.csv with
  the 161-var union schema, applying the AcroForm > MISC > HTML >
  Text precedence at each cell.
